## Notable findings that didn't make the top 5

- **Secrets in /tmp**: `install.sh:269` backs up the user's `.env` (may contain
  `GITHUB_TOKEN`) to world-readable, predictably-named `/tmp/brainstem-upgrade-$$`
  during upgrades. Should use `mktemp -d` with 700 perms.
- **Version pins are not sticky**: a no-arg installer re-run hits
  `git reset --hard origin/main` (`install.sh:302`), silently yanking pinned/rolled-back
  users to main.
- **install.ps1 parity gaps**: no venv (pip installs pollute user site-packages), no
  `--version` pinning/rollback, no port-7071 kill before launch. (Pip bootstrap and
  UTF-8 encoding ARE fixed on main as of 0.6.2/0.6.3.)
- **`gho_` tokens are dropped** by `get_github_token` (intended for Copilot), which is
  why the Get Help API path 403s for gh-CLI users and everything routes through the CLI
  fallback.
- **Test suite reality**: 24/24 pass — but only under `~/.brainstem/venv/bin/python`
  (system python3 lacks flask; the CLAUDE.md `python3 -m pytest` command misleads).
  `TestMemoryAgentIntegration` requires network (no skip guard). Untested hot spots:
  `call_copilot` (timeouts, fallback loop, SSE-to-be), `/diagnostics/report`, token
  refresh.
- **Preflight exists and is good** (`.github/workflows/preflight.yml` + `RELEASING.md` +
  `tests/preflight_local.sh`, added in the pearl pass) — plans above route through it.
- **Two distribution channels**: macOS one-liner + downloads serve from GitHub Pages
  (`docs/`), Windows one-liner serves from `raw.../main`. Root↔docs mirroring is manual;
  plan 3 adds a drift guard to `tests/test_installer.sh`.
- **Docs stale on removed features**: README.md:88/:147 and skill.md:151 still document
  the Sources panel / `/repos` endpoints (deleted in v0.1.0) — covered in plan 3.
